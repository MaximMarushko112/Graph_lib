# Графовые алгоритмы (C++)

Библиотека для работы с графами, реализующая три основных представления (список рёбер, список смежности, матрица смежности) и набор классических алгоритмов: обходы в ширину и глубину, поиск мостов и точек сочленения, алгоритмы поиска кратчайших путей (Дейкстра, двойной Дейкстра, Форд–Беллман, Флойд–Уоршелл).

## ✨ Возможности

- **Три типа графов**:
  - `EdgeList` – список рёбер
  - `AdjacencyList` – список смежности
  - `AdjacencyMatrix` – матрица смежности
- Поддержка направленных/ненаправленных, взвешенных/невзвешенных графов
- Шаблонный дизайн: вершины могут быть любого типа (обычно указатели)
- Итераторы с фильтрацией (например, обход только нечётных вершин)
- **Алгоритмы**:
  - BFS (поиск в ширину) с visitor-паттерном
  - DFS (поиск в глубину) с visitor-паттерном
  - Поиск мостов (`FindBridges`)
  - Поиск точек сочленения (`FindPointsOfArticulation`)
  - Кратчайшие пути:
    - Дейкстра (две версии: O(E log V) и O(V²), выбор по разреженности)
    - Двойной Дейкстра (для одной пары вершин)
    - Форд–Беллман (обработка отрицательных весов и циклов)
    - Флойд–Уоршелл (все пары вершин)
- Юнит‑тесты с использованием **Google Test**

## 🧪 Пример использования

```cpp
#include "graph types/adjacency_list.hpp"
#include "shortest paths/dijcstra_shortest_paths.hpp"

int main() {
    // Создаём ненаправленный взвешенный граф
    AdjacencyList<int, false, true, int> graph;
    
    int a = 1, b = 2, c = 3;
    graph.add_vertex(&a);
    graph.add_vertex(&b);
    graph.add_vertex(&c);
    
    graph.add_edge(&a, &b, 5);
    graph.add_edge(&b, &c, 3);
    graph.add_edge(&a, &c, 10);
    
    // Дейкстра от вершины a
    auto dist = DijcstraShortestPaths<decltype(graph), true>(graph, &a);
    // dist[&b] == 5, dist[&c] == 8
    return 0;
}
```

## 📖 Описание алгоритмов

### BFS / DFS

Реализованы через паттерн **Visitor**, позволяющий переопределить события:
- `discover_vertex`, `examine_edge`, `tree_edge`, `back_edge` и др.

Пример: подсчёт суммы значений вершин при обходе (как в тестах).

### Мосты и точки сочленения

- `FindBridges` – возвращает множество рёбер, удаление которых увеличивает число компонент связности.
- `FindPointsOfArticulation` – возвращает множество вершин, удаление которых увеличивает число компонент связности.

### Кратчайшие пути

| Алгоритм | Сложность | Отриц. веса | Отриц. циклы |
|----------|-----------|-------------|---------------|
| Дейкстра (E log V) | O((V+E) log V) | нет | нет |
| Дейкстра (V²) | O(V²) | нет | нет |
| Двойной Дейкстра | ~2×Дейкстра | нет | нет |
| Форд–Беллман | O(VE) | да | обнаруживает |
| Флойд–Уоршелл | O(V³) | да (нет циклов) | нет |

## 🔧 Особенности реализации

- Вершины хранятся как указатели (`VertexType*`) – это позволяет использовать любые пользовательские типы.
- Для хранения рёбер в списке рёбер и матрице используется специальная хеш-функция `EdgeHash`.
- В `AdjacencyMatrix` максимальное количество вершин задаётся при создании, поддерживается удаление вершин со сдвигом индексов (с помощью стека свободных индексов).
- Итераторы поддерживают **фильтры** (например, только вершины, удовлетворяющие предикату).

## 🧪 Тесты

Покрыты все три типа графов для каждого алгоритма:
- `EdgeListTest`, `AdjacencyListTest`, `AdjacencyMatrixTest` для BFS, DFS, мостов, точек сочленения, Дейкстры, Форда–Беллмана, двойного Дейкстры и Флойда–Уоршелла.
- Отдельные тесты на удаление вершин/рёбер, обработку исключений, работу итераторов.
